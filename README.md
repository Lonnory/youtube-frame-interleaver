# 🎬 YouTube Frame Interleaving (60 FPS / 2x Speed)

<br>
   
   [![Смотреть пример работы](https://img.youtube.com/vi/EUDr_UJUibU/maxresdefault.jpg)](https://youtu.be/EUDr_UJUibU)
   
   <br>
   
   ---
Этот скрипт позволяет объединить два разных видеофайла в один с помощью метода **Frame Interleaving** (чередования кадров). 
Суть эффекта заключается в том, что при загрузке итогового видео на YouTube:
- При просмотре в стандартном качестве (30 FPS) зритель будет видеть только **первое видео**.
- Если зритель переключит качество на 60 FPS или изменит скорость воспроизведения на 2x, он внезапно увидит **второе (скрытое) видео**. 
*(Важно: для корректного отображения скрытого видео монитор или экран зрителя должен работать строго на частоте 60 Гц).*

## 🧠 Как это работает?

YouTube часто сжимает видео до 30 кадров в секунду (отбрасывая каждый второй кадр), если пользователь не выбрал высокое качество. 
Скрипт создает 60 FPS видео, где:
- **Четные кадры** (0, 2, 4...) берутся из `Video1.mp4`.
- **Нечетные кадры** (1, 3, 5...) берутся из `Video2.mp4`.

## 🚀 Использование (готовый .exe)

Для запуска не обязательно иметь установленный Python. Достаточно скачать готовый `.exe` релиз (если доступен).

1. Поместите файлы `Video1.mp4` (Первое видео) и `Video2.mp4` (Второе видео) в одну папку с `frame_interleaver.exe`.
2. Запустите `frame_interleaver.exe`.
3. Скрипт проверит количество кадров. Если оно отличается, вам будет предложено обрезать более длинное видео.
4. Дождитесь окончания обработки (прогресс будет отображаться в процентах).
5. Готовый файл `OutVideo.mp4` можете выкладывавать на Youtube!

## ⚠️ Секреты записи и монтажа (ВАЖНО!)

Чтобы эффект сработал идеально и кадры не смешивались, строго соблюдайте следующие правила:

1. **Максимальные настройки записи**: Исходные видео (`Video1.mp4` и `Video2.mp4`) необходимо записывать на максимальном битрейте (например, 50 Mbps / 50000 Kbps в OBS) и строго в **30 FPS**. Чем чище пиксели и меньше артефактов сжатия, тем лучше скроется ваше секретное видео.
2. **Дисклеймер в начале**: Если вам нужно вставить дисклеймер на первые 5 секунд, делайте это в вашем видеоредакторе (например, в CapCut) **ДО** запуска этого скрипта. Создайте два проекта, вставьте абсолютно одинаковый дисклеймер в оба так, чтобы их длительность совпадала до единого кадра, и экспортируйте оба ролика в 30 FPS. Только после этого скармливайте их утилите. В противном случае в начале видео будет "киш-миш".
3. **Лайфхак для возврата всех разрешений на YouTube**: Чтобы у вашего итогового видео на YouTube было больше вариаций разрешений (например, появились 240p и 480p), нужно рендерить каждый исходный материал (`Video1.mp4` и `Video2.mp4`) в разрешении 2K (1440p) или 4K перед конвертацией в единое видео. Это заставит алгоритмы YouTube выдать более качественный кодек (VP09) и создать все варианты разрешений, так как сам скрипт не меняет разрешение видео, а сохраняет исходное.
4. **Запрет редактирования финала**: Полученный после скрипта файл `OutVideo.mp4` **КАТЕГОРИЧЕСКИ НЕЛЬЗЯ** редактировать, обрезать или повторно рендерить в CapCut, Premiere Pro или любом другом редакторе! Любой повторный рендер перемешает кадры, и эффект будет безвозвратно сломан. Загружайте файл на YouTube в его первозданном виде.

### Использование скрипта (Python)

Если вы хотите запустить исходный код или модифицировать его:

**Требования:**
- Python 3.7+
- OpenCV (`opencv-python`)

**Установка зависимостей:**
```bash
pip install opencv-python
```

**Запуск:**
Положите `Video1.mp4` и `Video2.mp4` рядом со скриптом и выполните:
```bash
python frame_interleaver.py
```

### Как скомпилировать в .exe самостоятельно

Вы можете легко упаковать этот скрипт в самостоятельный исполняемый файл для Windows с помощью `PyInstaller`.

1. Установите необходимые библиотеки:
```bash
pip install opencv-python pyinstaller
```
2. Запустите компиляцию:
```bash
pyinstaller --onefile --console frame_interleaver.py
```
3. Ваш готовый файл будет находиться в папке `dist/frame_interleaver.exe`.

## ⚙️ Технические ограничения

- **Ограничение экранов 60 Гц**: Эффект скрытого видео корректно работает только на классических мониторах и экранах смартфонов с частотой обновления 60 Гц. На игровых мониторах (144 Гц, 240 Гц и выше) или на смартфонах с экранами 90/120 Гц кадры будут хаотично смешиваться из-за особенностей рендеринга видео в Windows/Android/iOS. Зритель на таком экране может видеть мерцание или два видео одновременно. Лучше всего указать это в начале видео, чтобы пользователь сам поменял частоту экрана на 60Гц.
- **Задержка обработки YouTube**: После загрузки видео на YouTube алгоритмы платформы могут обрабатывать версию 1080p60/1440p60 (или выше) в течение нескольких часов. Сразу после загрузки эффект может не работать. Пожалуйста, проявите терпение и просто подождите, пока YouTube закончит полную обработку всех вариантов качества.
